╔════════════════════════════════════════════════════════════════════════════════╗
║                            API ENDPOINTS                                      ║
╚════════════════════════════════════════════════════════════════════════════════╝

GET /health
  └─ Returns: {"message": "Service is healthy"}

GET /recipes
  └─ Returns: List[Recipe]
  └─ Scans entire DynamoDB table

POST /recipes
  ├─ Request: Recipe object
  └─ Creates new recipe with UUID
  └─ Stores in DynamoDB

DELETE /recipes/{recipe_id}
  ├─ Parameter: recipe_id (String)
  └─ Deletes recipe from DynamoDB


╔════════════════════════════════════════════════════════════════════════════════╗
║                            DATA FLOW                                          ║
╚════════════════════════════════════════════════════════════════════════════════╝

1. USER ACCESSES FRONTEND
   User Browser → HTTPS → CloudFront → S3 Bucket → React App

2. USER CREATES RECIPE
   React App → HTTP → EC2 (FastAPI) → POST /recipes → DynamoDB

3. USER VIEWS RECIPES
   React App → HTTP → EC2 (FastAPI) → GET /recipes → DynamoDB Scan

4. USER DELETES RECIPE
   React App → HTTP → EC2 (FastAPI) → DELETE /recipes/{id} → DynamoDB


╔════════════════════════════════════════════════════════════════════════════════╗
║                            SECURITY LAYERS                                    ║
╚════════════════════════════════════════════════════════════════════════════════╝

✓ IAM Role: EC2 instance has limited DynamoDB permissions
  ├─ dynamodb:PutItem
  ├─ dynamodb:Scan
  └─ dynamodb:DeleteItem

✓ Security Group: Only HTTP (80) allowed from anywhere

✓ S3 Bucket Policy: CloudFront OAI for secure access

✓ Public Access Block: S3 bucket not publicly accessible

✓ CORS: Enabled on FastAPI for cross-origin requests
